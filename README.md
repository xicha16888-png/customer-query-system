# 客户交叉查询系统 v1.1

同时查询「手机分期 MORODOK」「小贷/抵押 PAWN」，并参考静态导入的历史客户名单，避免同一个客户重复借款。

## 这次更新了什么

在原来两个实时数据源基础上，加了第三个数据源：**历史客户名单**（`legacy-customers.json`，共7778条，
来自手机抵押老客户表，只有姓名/身份证号/性别/历史状态，没有金额和日期，纯参考用）。

- 查询页新增了"身份证号"输入框
- 结果里多一栏"历史名单参考"，标注该客户历史上是否有"锁机/逾期"记录
- `widget.js` 的 `confirmBeforeSubmit()` 现在支持传第三个参数身份证号，PAWN 那边的补丁已经同步更新

## 关于历史名单的说明

- 只用于"这个人以前来过、有没有不良记录"这种参考性提示，不代表真实未结清金额
- 状态分类：`已结清(历史)` / `锁机/逾期(历史)` / `其他备注(历史)`（原表里五花八门的备注，比如"add fb"、"lose phone"等，统一归到这类）/ `无标记(历史)`
- 只有"锁机/逾期"状态会触发醒目提示，其余仅展示
